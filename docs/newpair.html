How to Create New Pair

Create your own pair

A pre-stored pair is used in this procedure. Token contract should be instantiated in advance.

The JSON parameter for instantiation is below:

{
  "pair_code_id": "1",
  "token_code_id": "2",
  "init_hook": {
    "msg": "base64_encoded_json_data",
    "contract_addr": "secret..."
  }
}

The factory contract is responsible for creation of SecretSwap pairs as well as being the directory for all pairs. You may input numbers corresponding to pair_code_id and token_id. The token contract address is used for init_hook.contract_addr.

To set init_hook.msg, you must first organize another JSON as below:

{
  "create_pair": {
    "asset_infos": [
      {
        "token": {
          "contract_address": "secret..."
        }
      },
      {
        "native_token": {
          "denom": "uusd"
        }
      }
    ]
  }
}

This is a JSON constructor of pair contract.

  • A token pair can be either a contract-based token, or a secret-native token
    • asset_infos[x].token.contract_address: Contract-based token address is entered here.
    • asset_infos[x].native_token.denom: Secret native token denominator is entered here.
